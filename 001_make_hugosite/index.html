HUGO周りの扱い方

HUGOのインストール

HUGOを最初にインストールします。Ubuntuは非常に簡単です。

$sudo apt install hugo

フォルダ作成

HUGOを立ち上げるためのFolderを作ります。

$hugo new site {$new-site-name-folder}
$cd {$new-site-name-folder}

これで、新規フォルダーが完成していることを確認できる。

一時的にサーバー立てて練習

HUGOでサーバーを立てて練習できる。出来るのは、.configファイルがある場所のみ。

$hugo server

ただし、はじめは何も入れていないので、真っ白なページのみが表示される。

サイトに味付けをする

hugoは様々なテーマがあるので、それを入れる。

なんでも良いのだが、今回は気に入ったので”cupper”というテーマを入れてみよう。

hugo_theme_Cupper

と思ったら、最低でも0.54が必要らしい。ので、至急入れ直した。

hugoの最新バージョンはここにあったので、適宜あったもの（OS,解凍方法,アーキテクチャなど）を入れる。

あとは、Cupperの指示に従った。

$cd {$hugo_setting.config_folder}
$git clone https://github.com/zwbetz-gh/cupper-hugo-theme.git themes/cupper-hugo-theme

この操作によって、Themeのフォルダにテーマを一つ入れることができた。

テーマ・URLの設定

ここまで編集すると、実際にできているか気になるので、

$hugo server --ThemesDir ../..

とすると実際のデモを見ることができた。ただ、いちいちこんなことを書くのは面倒なので、ファイルを編集しなおす。

$nano config.toml

=== content ===
    # ここは適宜
    languageCode = "en-us"
    title = "My New Hugo Site"

    # githubのURLにしてください
    baseURL = "http://example.org/"

    # 今回の登録テーマのフォルダ名を入れれば大概大丈夫です。
    theme = "cupper-hugo-theme"

    # なんだこれ！？
    canonifyurls = true

=== END ===

hosting contentsの生成

$hugo

と打ち出すと、編集した内容をもとにpublicにデータを生成する。このパブリックをgithubに追跡させる。

$git init
$git remote add origin git@github.com:***/*******.git
$git checkout -b gh-pages
$git add --all
$git commit -m'***'
$git push origin gh-pages

まあ、このへんはGitの操作してたら当然のことだらけだけど。

そうして、githubのページ設定から、すぐサイトを見に行くことができる。そうすると、上手く反映されていなかった…（これは色々テーマの設定を忘れていたから。）

これでサイトを書き換えたらすぐにhugoと打つことで反映され、またサイトも更新可能になる。

hugoのmetaデータ記述とmdのデータ格納

ここからはちょっと細かいことについて。

hugoはmdファイルについてメタデータを打つことが出来る。また、その書き方は色んなとこに書いているからそれを参照するが、その書き方を最初に指定できるのが*archtypes*というフォルダのアイテムである。

おおよそmdの記事はタグ付けしたいと思うし、日付もほしい。そういうときに便利な道具たちだと思う。

ただし、注意してほしいのがdraftである。これはまだ草案であることを明示的にする要素であり、この要素がtrueだと、hugoで作成されないので、完成したら外そう。

ただ、その状態でもテストしたいと思うときは、

$hugo server -D

で表示できます。

ファイル構造について

ファイル構成は、自分の使っているCupperの形式では

  • content
    • _index.md
    • {your_folder}
      • 貴方のマークダウンの記事1
      • 貴方のマークダウンの記事2
      • 貴方のマークダウンの記事3
      • …
    • {your_other_folder}
      • 貴方のマークダウンの記事1
      • 貴方のマークダウンの記事2
      • 貴方のマークダウンの記事3
      • …

のような形でなければ悪かった。contentを読むようになっているらしいので、外部にpostフォルダを作っても認識されなかった。
